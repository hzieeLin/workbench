# 今日聚焦与默认看板结构设计

## 1. 背景与目标

当前核心看板已支持看板、列表和任务卡片管理，以及搜索、列表视图、日历视图、任务待办和评论。产品近期主动移除了标签、复杂筛选、排序和时间线等能力，因此本次设计继续保持轻量，重点解决两个效率问题：

1. 新看板为空，用户每次都要重复创建基础工作流。
2. 看板能保存任务，但缺少“今天先做什么”的明确引导。

本次采用“聚焦闭环”方案：新建看板自动获得基础三列表，并在看板顶部提供每日聚焦区。系统根据截止日期给出建议，用户可以按当天实际安排调整。

## 2. 范围

### 2.1 本期包含

- 新建看板时自动创建 `待开始`、`进行中`、`已完成` 三个列表。
- 看板顶部增加可折叠的“今日聚焦”横栏。
- 自动纳入今天到期和已经逾期的未完成任务。
- 用户可以手动加入或移出当天聚焦任务。
- 聚焦任务超过 3 项时给出柔性提示，但不阻止添加。
- 在聚焦栏内汇总逾期任务数量。
- 任务进入名为 `已完成` 的列表时退出聚焦区。
- 跨日、应用恢复和重新进入看板时刷新当天聚焦结果。

### 2.2 本期不包含

- 重复任务。
- 通用自动化规则。
- 专注计时。
- 批量操作。
- 系统桌面通知。
- 新的优先级或截止日期筛选。
- 为列表增加固定状态类型。
- 自动修改已有看板的列表结构。

## 3. 核心行为

### 3.1 新看板默认列表

创建新看板时，服务端在同一个数据库事务中执行以下操作：

1. 创建看板。
2. 创建 `待开始` 列表，位置为 1。
3. 创建 `进行中` 列表，位置为 2。
4. 创建 `已完成` 列表，位置为 3。

任一步失败时回滚整个事务，不能留下没有默认列表的看板或孤立列表。创建成功后，接口返回新看板；前端选中该看板，并按现有加载流程取得三个列表。

已有看板保持原样。用户仍可新增、重命名、排序或删除默认列表。

### 3.2 完成语义

本期不为列表增加状态字段。读取聚焦任务时，如果任务所在列表的名称去除首尾空格后等于 `已完成`，该任务视为完成，不出现在聚焦区。

- 如果有多个名为 `已完成` 的列表，其中的任务都视为完成。
- 列表重命名后立即按新名称重新判定。
- 将 `已完成` 改成其他名称后，其中到期或逾期的任务可能重新进入聚焦区。这是名称判定方案的明确行为。
- 本规则只用于本期的聚焦计算，不在本期重构统计模块的完成口径。

### 3.3 今日聚焦计算

“今天”使用运行服务所在环境的本地自然日，以 `YYYY-MM-DD` 表示。不能通过截断 UTC 时间得到本地日期。

某任务进入今日聚焦区的计算顺序如下：

1. 任务位于当前看板内。
2. 任务不在名称为 `已完成` 的列表中。
3. 如果当天存在手动排除记录，则不纳入。
4. 否则，如果当天存在手动加入记录，则纳入。
5. 否则，如果任务截止日期早于或等于今天，则自动纳入。
6. 其他任务不纳入。

手动加入可以作用于没有截止日期或截止日期在未来的任务。手动加入和排除只对当天有效；第二天重新按新日期和自动规则计算。

聚焦任务按以下顺序展示：

1. 已逾期任务，截止时间从早到晚。
2. 今天到期任务，截止时间从早到晚。
3. 手动加入的其他任务，按手动加入时间从早到晚。

排序只影响聚焦栏，不改变任务在看板列表中的位置。

### 3.4 三项柔性建议

聚焦任务数量不受硬限制。当数量超过 3 项时，聚焦栏显示轻量提示：`建议保留最重要的 3 项`。提示不弹窗、不阻断加入操作，也不自动移除任务。

## 4. 数据设计

新增 `DailyFocusOverride` 实体及 `daily_focus_overrides` 表：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | 自增整数 | 主键 |
| `card_id` | 整数 | 关联任务 |
| `focus_date` | date | 生效的本地自然日 |
| `mode` | text | `include` 或 `exclude` |
| `created_at` | datetime | 创建时间，也用于手动任务排序 |
| `updated_at` | datetime | 最近修改时间 |

约束与清理规则：

- `card_id + focus_date` 建立唯一约束。
- 同一天重复设置时使用更新而不是插入重复记录。
- 删除任务时级联删除其聚焦覆盖记录，或在任务删除流程中显式清理。
- 历史覆盖记录不影响当前结果，可在后续维护任务中按保留策略清理；本期不增加用户可见的历史界面。

## 5. 接口设计

### 5.1 创建看板

保留现有 `POST /boards` 请求格式。服务端响应成功前完成看板和默认列表的事务创建。

### 5.2 读取今日聚焦

`GET /boards/:boardId/focus?date=YYYY-MM-DD`

- `date` 由前端按本地日期传入，用于跨日刷新和可测试性。
- 服务端校验日期格式，不接受任意时间字符串。
- 返回聚焦任务、来源、是否逾期和汇总数量。
- 服务端仍需按任务所属列表校验任务确实属于该看板。

建议响应：

```json
{
  "date": "2026-06-18",
  "items": [
    {
      "card": {},
      "source": "automatic",
      "overdue": true
    }
  ],
  "total": 1,
  "overdueCount": 1
}
```

`source` 为 `automatic` 或 `manual`。任务即使原本符合自动规则，只要当天被用户显式加入，仍可保留其最终覆盖状态；界面不需要突出这一内部差异。

### 5.3 设置当天状态

`PUT /cards/:cardId/focus`

```json
{
  "date": "2026-06-18",
  "mode": "include"
}
```

`mode` 只能为 `include` 或 `exclude`。移出自动纳入的任务写入 `exclude`；加入其他任务写入 `include`。服务端使用唯一键进行新增或更新，并返回该任务当天的最终聚焦状态。

## 6. 前端组件与交互

### 6.1 `TodayFocusBar`

组件位于看板标题区与搜索栏之间，仅在已选中看板时显示。默认展开，折叠状态按本机保存，不写入数据库。

展开状态包含：

- 标题“今日聚焦”。
- 当前数量和逾期数量。
- 紧凑任务卡片列表。
- “选择任务”入口。
- 超过 3 项时的柔性提示。
- 折叠控制。

点击聚焦任务打开现有任务详情。每项提供移出当天聚焦的操作，该操作不会删除或移动任务。

空状态显示：`今天还没有聚焦任务`，并保留“选择任务”入口。

### 6.2 普通任务卡与详情

- 普通任务卡悬停时显示“加入今日聚焦”或“移出今日聚焦”的快捷操作。
- 任务详情中提供相同操作，作为可发现性更高的稳定入口。
- 操作按钮必须阻止冒泡，避免切换聚焦状态时同时打开任务详情。
- 任务移入 `已完成` 列表后，前端重新拉取或本地同步聚焦结果，使任务立即从聚焦栏消失。

### 6.3 “选择任务”入口

“选择任务”打开轻量选择面板，展示当前看板中未完成且尚未聚焦的任务。支持按现有任务标题搜索，但不引入新的复杂筛选。选择任务后写入当天 `include` 记录。

## 7. 状态流与刷新

1. 用户进入或切换看板。
2. 前端加载列表和任务，同时请求当日聚焦数据。
3. `TodayFocusBar` 独立显示加载状态，不阻塞主看板。
4. 用户加入或移出任务后，服务端确认成功，再更新聚焦状态。
5. 用户移动任务进入或离开 `已完成` 列表后，刷新聚焦计算。
6. 应用从休眠恢复、窗口重新获得焦点或本地日期发生变化时，比较当前日期；若跨日则加载新一天数据。

前端以服务端返回结果为准，不使用无法回滚的乐观更新。

## 8. 错误与边界处理

- 建板事务失败：不返回成功，不保留部分数据；前端保留输入并提示重试。
- 聚焦加载失败：主看板继续可用；聚焦栏显示“加载失败，重新加载”。
- 聚焦写入失败：保持原状态并显示轻量错误提示。
- 无效日期或 `mode`：接口返回 400。
- 看板、任务不存在：接口返回 404。
- 任务不属于指定看板：读取时不泄漏或混入其他看板任务。
- 没有截止日期：不自动纳入，但允许手动加入。
- 截止时间恰好在今天：按本地自然日视为今天到期。
- 多个同名完成列表：统一按完成处理。
- 删除列表或任务后：聚焦结果不得保留不可打开的悬空项。

## 9. 测试策略

### 9.1 服务端与数据层

- 创建看板成功时同时生成三个默认列表，名称和位置正确。
- 默认列表任一写入失败时完整回滚。
- 已有看板不受迁移或新建逻辑影响。
- 今天到期、已逾期和未来任务的自动规则正确。
- 手动加入无截止日期或未来任务。
- 手动排除自动任务。
- 当天覆盖记录不影响次日。
- 名为 `已完成` 的列表及首尾空格判定正确。
- 唯一约束阻止同任务同日期重复记录。
- 删除任务后不残留有效聚焦项。

### 9.2 组件与状态层

- 聚焦栏加载、空、失败、折叠和正常状态。
- 逾期数量与任务总数正确。
- 超过 3 项显示提示，但第 4 项仍能加入。
- 卡片快捷操作不会误触任务详情。
- 移入 `已完成` 后任务从聚焦栏移除。
- 跨日和应用恢复触发重新加载。

### 9.3 端到端流程

覆盖以下主路径：创建看板并看到三个默认列表；创建今天到期任务并自动进入聚焦区；手动加入另一任务；排除逾期任务；将任务移入 `已完成` 并确认其退出聚焦区。

## 10. 验收标准

1. 每个新看板创建后立即具有顺序正确的三个默认列表，且不存在半成功状态。
2. 用户进入看板后能在顶部直接看到当天到期和逾期任务。
3. 用户能在不移动或删除任务的情况下调整当天聚焦项。
4. 聚焦超过 3 项只提示，不限制用户操作。
5. 应用内能看到逾期汇总，但不会产生系统桌面通知或打断式弹窗。
6. 位于名为 `已完成` 的列表中的任务不出现在聚焦区。
7. 手动聚焦调整只影响当天，跨日后按新一天重新计算。
8. 聚焦功能故障不会阻塞看板的核心浏览和编辑流程。
